# Handoffs

Context-passing documents for complex multi-session work. When a task spans multiple sessions or agents, write a handoff so the next session can pick up where you left off.

## What goes here

- Session handoffs (what was done, what's next, what to watch out for)
- Agent-to-agent context (when one specialist hands off to another)
- Multi-day task continuations

## Format

Name files with date prefix: `2026-04-07-auth-refactor.md`

Each handoff should include:
- **What was done** — completed work
- **What's next** — specific next steps with file paths
- **Watch out for** — gotchas, known issues, things that almost broke
- **Verification** — how to confirm the current state is correct before continuing
