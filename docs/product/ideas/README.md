# Ideas

Half-baked thoughts worth capturing before they evaporate. Lighter than a feature proposal.

## What goes here

- "What if we…" sketches
- Observations that might matter but you haven't validated
- Connections between things you noticed
- Random sparks from reading / talking to users / using competitors

## File naming

Whatever's easy. Suggestions:
- One file per idea: `<slug>.md` (if the idea is substantive).
- Or one running list: `ideas.md` with date-stamped entries (if most ideas are one-liners).

## Format

Keep it low-friction — this is a capture tool, not a deliverable. Minimum:

```markdown
# <Idea title>

- **Captured:** YYYY-MM-DD

<2-5 sentences on what + why you thought of it>

## Why it might matter
<1-2 sentences>

## Why it might not
<1-2 sentences>
```

## Lifecycle

- **Promote to feature proposal** (`../features/`) when the idea has enough shape to describe a problem and solution.
- **Archive or delete** after ~6 months of no movement. If it didn't graduate, it probably wasn't it.
- **Link to it from a decision record** if it ever becomes the seed of something shipped — provenance matters.

## Why keep these at all?

Two reasons:
1. Getting it out of your head makes room for the next thought.
2. Reading old ideas months later occasionally sparks something you couldn't have planned.
