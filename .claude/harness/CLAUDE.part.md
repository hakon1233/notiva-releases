# Harness skills (auto-managed, shared across every project)

## Skill + Agent invocation is NOT optional

Skill descriptions in your `skill_listing` and the table below are **directives**, not suggestions. Even if a task seems self-contained and you think you can solve it directly:

- **When a skill description's trigger condition matches your task, you MUST invoke `Skill('name')` BEFORE acting.** The body contains context, conventions, and gotchas the description doesn't show. Skipping = working without project knowledge.
- **Same for agents** — when an agent's trigger matches, dispatch via `Agent(subagent_type='...')` instead of doing the agent's job yourself.
- **"This bug looks simple, I don't need `test-first`" is the wrong heuristic.** The skill exists because past sessions also thought that and shipped broken code. Invoke it.
- Exception: if you've **already loaded** the skill earlier in this session, you can act on its content directly without re-loading.

If you're about to call Read/Edit/Bash on a task whose trigger matches a skill in the table below, stop and call `Skill('name')` first.

---

Claude Code auto-loads skills based on their frontmatter description. The full set ships with every project as part of the harness version stamp. Do not edit these rules here — see the template in task-terminal-manager to propose changes, or override for your project by adding instructions in `.claude/project/CLAUDE.part.md`.
